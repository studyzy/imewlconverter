## 新增需求

### 需求：批量调用 LLM 生成词频
系统必须支持将一批词语（如每批 50 个）合并发送给 OpenAI 兼容的 Chat Completion API，以获取这些词语的词频。

#### 场景：成功批量获取词频
- **当** 用户请求为一批词语生成词频
- **那么** 系统向 LLM API 发送包含这些词语的请求，并解析返回结果，将其中的数字映射回对应的词语

### 需求：内置优化提示词
系统必须内置经过优化的提示词（Prompt），不再要求用户手动输入。内置提示词应要求 LLM 返回易于解析的格式（如 JSON 或纯数字列表）。

#### 场景：使用内置 Prompt 批量请求
- **当** 系统构建请求时
- **那么** 系统必须使用内置的提示词模板，并将该批次的词语填充进去
