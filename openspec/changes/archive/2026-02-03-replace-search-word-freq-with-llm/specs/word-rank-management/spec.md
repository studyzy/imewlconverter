## 移除需求

### 需求：百度词频生成器
**Reason**: 百度搜索结果解析不稳定，且容易触发反爬虫机制。
**Migration**: 用户应当切换到新增的 LLM 词频生成器。

### 需求：谷歌词频生成器
**Reason**: 谷歌搜索结果解析依赖于特定的 HTML 结构，维护成本高且访问受限。
**Migration**: 用户应当切换到新增的 LLM 词频生成器。

## 修改需求

### 需求：选择词频生成来源
系统必须允许用户在可用来源中选择一个用于生成词频。

#### 场景：选择 LLM 作为来源
- **当** 用户在词频生成窗口的下拉列表中选择“LLM 生成”
- **那么** 系统必须在后续操作中使用 LLM 接口获取词频
